# Diploma_work_Python
## Дипломная работа. Архитектура фреймворка. Автоматизация тестирования проекта "Читай-город"

### Описание проекта
В рамках данного проекта реализован комплекс автоматизированных тестов для интернет‑магазина «Читай‑город». Проверки реализованы как на уровне API, так и на уровне пользовательского интерфейса (UI). Тестируются основные сценарии: работа с корзиной (добавление, изменение и удаление позиций) и функционал поиска.

### Описание классов
- **AddToCartAPI**: класс, реализующий взаимодействие с API для добавления товаров в корзину. 
- **IncrementCartItemQuantityApi**: класс, предназначенный для взаимодействия с API с целью увеличения количества товара в корзине пользователя.
- **WrongMethodAddToCartApi**: класс, имитирующий некорректное взаимодействие с API добавления товара в корзину.
- **AddToCartInvalidUrlApi**: класс, имитирующий ошибочное обращение к API добавления товара в корзину через некорректный URL.
- **ClearCartApi**: класс предназначен для взаимодействия с API с целью полной очистки корзины пользователя (удаления всех товаров).
- **MainPage**: класс, предназначенный для тестирования функционала поиска книг по названию на сайте «Читай‑город». Проверяет корректность работы поискового механизма при вводе валидного названия книги.
- **MainPageInvalidName**: класс предназначен для тестирования устойчивости поля поиска по названию книги на сайте «Читай‑город» при вводе некорректных, недопустимых или заведомо ошибочных данных. Позволяет проверить, как система обрабатывает пограничные и нештатные сценарии ввода.
------
- **test_api.py**: файл с тестовыми сценариями для API, покрывающий операции управления корзиной (добавление, удаление, обновление товаров), а также проверку обработки некорректных запросов.
- **test_ui.py**: файл, который содержит набор тестовых сценариев для проверки пользовательского интерфейса. В рамках тестирования анализируется работа полей поиска при вводе как корректных (валидных), так и некорректных (невалидных) данных.
------
### Описание базового синтаксиса записи и форматирования

Проект использует следующие технологии и синтаксис:

1. **Python**: основной язык программирования для написания тестов.
2. **Selenium**: библиотека для автоматизации взаимодействия с веб-браузером.
3. **Pytest**: фреймворк для написания и запуска тестов.
4. **Requests**: библиотека для отправки HTTP‑запросов и взаимодействия с веб‑сервисами/API.
5. **Allure**: инструмент для генерации отчетов о выполнении тестов.

### Форматирование кода

- Код форматируется в соответствии с PEP 8 (стиль написания кода на Python).
- Используются docstrings для документирования методов и функций.
- Все шаги теста размечаются с помощью `with allure.step` для улучшения читаемости отчетов.

### Получение токена авторизации и ID товара
Для работы с API вам понадобятся токен авторизации и ID товара:
откройте инструменты разработчика в браузере (клавиша F12). Перейдите на сайт Читай-город и выполните необходимые операции (например, добавление товара в корзину). На вкладке Network найдите запросы, отправляемые при добавлении товара в корзину. В заголовках запроса найдите поле Authorization. Скопируйте токен вместе со словом Bearer и вставьте его в файл Configuration.py. Чтобы найти ID товара, проверьте параметры запроса или ответ сервера. Обычно это поле id объекта товара. Вставьте его значение в файл test_api.py в @pytest.mark.parametrize для test_increment_cart_item_quantity и test_clear_cart.
**ВАЖНО**: обновление токена и ID товара нужно произвести до запуска тестов!

### Инструкция по запуску тестов
Запуск тестов API
Чтобы запустить тесты API, используйте следующую команду: `pytest -m api`

Запуск тестов UI
Чтобы запустить тесты пользовательского интерфейса, используйте следующую команду: `pytest -m ui`

Запуск всех тестов
Чтобы запустить все тесты в проекте, используйте следующую команду: `pytest`

### Инструкция по запуску тестов для формирования отчета Allure

1. Установите необходимые зависимости:
   `pip install -r requirements.txt`
2. Запустите тесты с генерацией отчета Allure:
   `python -m pytest --alluredir allure-result`
В директории с тестами появится папка `allure-result`. Там сохранятся отчеты о тестах.

### Инструкция по просмотру сформированного отчета Allure
После завершения тестов, чтобы просмотреть отчет, выполните команду `allure serve allure-result`
Эта команда запустит локальный сервер и откроет отчет в браузере.
